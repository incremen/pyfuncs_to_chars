pyfuncs_to_chars

represent any character using only python builtin function calls

The constraint: write a Python expression that evaluates to any character, using only builtin function calls. No numeric literals, no string literals, no operators. Each function takes exactly one argument.

So chr(97) is out (uses a number). ord('a') is out (uses a string). But ord(max(str(bytes()))) is fine — it evaluates to 98 using only function calls on function calls.

The question is: can you reach every Unicode character this way? And if so, how short can the expression be?

We need two operations. Together they can build any number.

Subtract 1: max(range(n)) returns n - 1.

range(n) produces 0, 1, ..., n-1. max() picks the last one. Costs 2 parentheses.

Multiply by 3: len(str(list(bytes(n)))) returns exactly 3n.

bytes(n) creates n zero-bytes. list() turns it into [0, 0, ..., 0]. str() gives "[0, 0, ..., 0]" — always exactly 3n characters. Costs 4 parentheses.

The algorithm: to represent n, decompose it in base 3. At each step, build ceil(n/3), triple it, then subtract the remainder (0, 1, or 2). Stop when you reach a base anchor — a small number you can construct directly (like len(str(not())) = 4). 

function build(n):
    if n is a base anchor:
        return the anchor expression

    q = ceil(n / 3)
    r = 3 * q - n              // r is 0, 1, or 2

    expr = triple(build(q))    // multiply by 3
    expr = subtract(expr, r)   // subtract 0, 1, or 2
    return expr

Example: build(13)

build(13):
    q = ceil(13/3) = 5,  r = 15 - 13 = 2
    13 = triple(build(5)) - 2

  build(5):
      q = ceil(5/3) = 2,  r = 6 - 5 = 1
      5 = triple(build(2)) - 1

    build(2):
        2 is a base anchor
        return len(str(ord(min(str(not())))))

  working back up:
    build(5) = triple(len(str(ord(min(str(not())))))) - 1
    build(13) = triple(that) - 2

Then wrap the whole thing in chr() to get the character.

Python has a 200 nested parentheses limit. The base-3 formula stays well under that for all Unicode code points (max 1,114,111).

The base-3 formula works for everything but isn't always the shortest. These strategies produce shorter expressions for specific numbers. The optimizer tries all of them on every number and keeps the best result.

Multipliers

Different ways to grow a number, each with a different parenthesis cost:

len(str(list(bytes(n))))= 3n4 parens
len(str(bytes(n)))= 4n + 33 parens
len(ascii(str(bytes(n))))= 5n + 54 parens

4n+3 costs only 3 parens (cheaper than 3x at 4), so it's often better when you can land on the right value with the +3 offset.

Zip chain

Each zip() wrapper adds 3n to the string length. Formula: len(str(list(zip...zip(bytes(n))...))) = 3(k+1)n

k=16n5 parens
k=29n6 parens
k=312n7 parens
k=415n8 parens

Exact multipliers with no offset. Good for hitting exact multiples of 6, 9, etc.

Ascii exponential

ascii() escapes backslashes, roughly doubling them each time. Formula: f(n) = (2^k + 3)n + (2^(k+1) + 1)

k=15n + 54 parens
k=27n + 95 parens
k=311n + 176 parens
k=419n + 337 parens
k=535n + 658 parens
k=667n + 1299 parens
k=101027n + 204913 parens

Exponential multiplier for linear paren cost. A 1027x multiplier in 13 parens.

Other

sum(range(n)) = n(n-1)/2 — quadratic jump, 2 parens.

len(str(n)) = digit count — collapses big numbers back down, 2 parens.

int(bool(n)) = 1 for n>0 — reset to 1, 2 parens.

The database stores the shortest known expression for each number from 0 to 200,000. Each entry records which strategy produced it and which smaller number it depends on.
