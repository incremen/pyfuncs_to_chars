pyfuncs_to_chars

represent any character using only python builtin function calls

how it works

The constraint: write a Python expression that evaluates to any character, using only builtin function calls. No numeric literals, no string literals, no operators. Each function takes exactly one argument.

We need two things: a way to multiply by 3, and a way to subtract 1. Together they let us build any number in base 3.

Subtract 1: max(range(n)) returns n-1.

range(n) produces 0, 1, ..., n-1. max() picks the last one.

Multiply by 3: len(str(list(bytes(n)))) returns exactly 3n.

bytes(n) creates n zero-bytes. list() turns it into [0, 0, ..., 0]. str() gives "[0, 0, ..., 0]" which is always exactly 3n characters long.

The algorithm: to build any number n, express it in base 3. At each step, recursively build ceil(n/3), triple it, then subtract 0, 1, or 2. The recursion bottoms out at a base anchor — a small number we can construct directly from builtins (like len(str(not())) = 4). 

# example: build 13
# 13 in base 3 is 111 (1*9 + 1*3 + 1)
# but we work top-down:

build(13):
  ceil(13/3) = 5,  remainder = 3*5 - 13 = 2
  so 13 = 3 * build(5) - 2

build(5):
  ceil(5/3) = 2,  remainder = 3*2 - 5 = 1
  so 5 = 3 * build(2) - 1

build(2):
  base anchor: len(str(ord(min(str(not())))))

# result for 13:
# triple(triple(anchor_2) - 1) - 2
# = len(str(list(bytes(max(range(len(str(list(bytes(
#     len(str(ord(min(str(not())))))
#   ))))))))))  ...and so on

Python has a 200 nested parentheses limit. The base-3 approach uses at most ~114 for the largest Unicode code point (1,114,111), so it fits.

optimizations

The base-3 approach works but isn't always the shortest. These strategies can produce shorter expressions for specific numbers.

Different ways to multiply, each with a different cost in parentheses:

len(str(list(bytes(n))))= 3n4 parens
len(str(bytes(n)))= 4n + 33 parens
len(ascii(str(bytes(n))))= 5n + 54 parens

The 4n+3 multiplier costs only 3 parens (cheaper than 3x at 4 parens), so it's often better when you can land on the right value with the +3 offset.

ascii() escapes backslashes, roughly doubling them each time. Stacking k calls of ascii() on str(bytes(n)) gives:

f(n) = (2^k + 3) * n + (2^(k+1) + 1)

k=15n + 54 parens
k=27n + 95 parens
k=311n + 176 parens
k=419n + 337 parens
k=535n + 658 parens
k=667n + 1299 parens
k=101027n + 204913 parens

Exponential multipliers for linear paren cost. A 1027x multiplier in 13 parens means reaching large numbers quickly.

Wrapping bytes(n) in zip() creates nested tuples, making the string representation longer. Each zip adds exactly 3n to the length:

len(str(list(zip^k(bytes(n))))) = 3(k+1) * n

k=03n4 parens (same as triple)
k=16n5 parens
k=29n6 parens
k=312n7 parens
k=415n8 parens

Clean exact multipliers with no offset terms. Useful when you need to hit a number that's an exact multiple of 6, 9, 12, etc.

Triangular: sum(range(n)) = n(n-1)/2. Quadratic jump, 2 parens. Good for getting from small seeds to medium numbers quickly.

Log step-down: len(str(n)) = number of digits. Collapses a huge number back to something small. 2 parens.

Boolean collapse: int(bool(n)) = 1 for any n > 0. Instant reset to 1. 2 parens.

List range repr: len(str(list(range(n)))) grows as ~n*log(n). Sub-quadratic, useful for controlled jumps into the tens of thousands. 4 parens.
